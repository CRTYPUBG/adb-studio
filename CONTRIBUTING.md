# Contributing

Read the architecture documentation and related ADRs before editing. Search for existing contracts
and implementations before adding files. Keep QML presentation-only and use FluentWinUI3 controls.

All changes require tests, documentation, resource-manifest updates and clean CI. Compiler warnings
are errors. Public interfaces require contract tests; architecture changes require an ADR. Run the
commands in `README.md` and summarize results in `Validation.md` and `Handoff.md`.
