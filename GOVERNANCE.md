# Governance

ADB Studio uses protected branches, mandatory CI and CODEOWNERS review. Maintainers approve
releases and security fixes. Architecture owners approve ADRs and dependency-boundary changes.
Security owners may embargo vulnerability details until a coordinated release is available.

No milestone is complete with failed checks, warnings, stale documentation, unreviewed dependency
changes or unresolved architecture violations. Decisions and review evidence are stored in the
repository. Project governance changes require maintainer consensus and a documented decision.
